## 1. 技术栈要求 (Tech Stack)
- 后端框架：Golang (异步支持)
- 数据库：SQLite (开发环境) / PostgreSQL (生产就绪) ，使用 SQLAlchemy ORM
- 前端框架：Vue 3 + Element Plus (或 React + Ant Design)
- 认证方式：JWT (用于管理员后台登录)

## 2. 核心功能模块 (Core Features)

### 模块一：后台自动接入与配置系统
- 目标：管理员可后台配置各大模型厂商（如 OpenAI, Anthropic, 百度千帆, 阿里云）的API Key。
- 关键逻辑：系统需模拟“自动登录”或“密钥验证”，点击“测试连接”时，系统自动调用该厂商的`/v1/models`接口，验证Key有效性并拉取模型列表。
- 数据存储：存储加密后的API Key、Base URL、模型名称及计费单价（$ / 1M Tokens）。

### 模块二：分销推广链接生成器
- 目标：针对每个已激活的模型，系统自动生成唯一的推广分销链接（或邀请码）。
- 支持：管理员可手动填写/覆盖推广链接的URL参数（如 `?ref=admin` 或 `?invite=xxx`）。
- 逻辑：当用户通过该链接访问时，系统需记录 `click_count` 和 `referrer`。

### 模块三：首页价格对比看板 (核心展示)
- 目标：这是网站首页，必须展示以下内容：
    1. 全部已接入的模型厂商 Logo/名称。
    2. **价格对比表**：以表格形式列出 Input Price (输入价格) 和 Output Price (输出价格)，单位统一为 `$/1M tokens`。
    3. **性能指标**：如果有公开的 ELO 评分或 Latency (延迟) 数据，可一并显示。
    4. 调用接口：前端从后端 `/api/v1/models/` 接口获取实时数据（需包含缓存机制，减少厂商API调用）。

## 3. 数据库模型设计建议 (Schema)
请帮我生成对应的 SQLAlchemy 模型代码：
- `Vendor` 表：id, name, official_url, api_base, auth_type
- `Model` 表：id, vendor_id, model_name, context_length, input_price, output_price, is_active
- `DistributionLink` 表：id, model_id, link_code, custom_url, clicks, created_at

## 4. 关键业务逻辑要求 (难点)
- **智能缓存**：价格数据不能每次访问都去请求厂商API（会被限流）。请设计一个 `@cached(ttl=3600)` 装饰器，优先读取数据库，后台点击“刷新”时才强制回源。
- **安全性**：API Key 必须在数据库中使用 `Fernet` (对称加密) 加密存储，前端严禁明文传输。
